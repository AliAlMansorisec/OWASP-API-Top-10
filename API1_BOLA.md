# 🌐 API1:2023 - Broken Object Level Authorization (BOLA)

---

## 📌 1. التعريف الاحترافي (Definition)

تحدث هذه الثغرة عندما لا يتحقق الـ API من أن المستخدم الذي يطلب "كائن" معين (Object) لديه الصلاحية للوصول إليه. يعتمد الـ API غالباً على معرف الكائن (ID) الذي يرسله المستخدم في الرابط (URL)، وإذا لم يكن هناك فحص دقيق، يمكن للمستخدم الوصول لبيانات غيره بمجرد تغيير الـ ID.

---

## 🧠 2. مثل واقعي (Real-World Analogy)

تخيل أنك ذهبت إلى **المطعم** 🍔:

1. طلبت وجبتك، وأعطاك الموظف **رقم طلب (7)** وقال لك: "انتظر حتى يجهز طلبك".
    
2. أنت جلست على الطاولة، وشفت ورقة صغيرة مكتوب عليها رقم طلبك **(7)**.
    
3. فجأة، خطرت لك فكرة! مسحت الرقم **(7)** وكتبت بدلاً منه **(8)**.
    
4. ذهبت للموظف وقلت له: "أعطني هذا الطلب".
    
5. الموظف نظر للورقة، ورأى رقم **(8)**، فذهب وأحضر لك وجبة الشخص الآخر (الذي طلب رقم 8) وأعطاك إياها **بدون ما يتأكد من هويتك أو يطلب منك الفاتورة الأصلية!**

👉 **هذه هي ثغرة BOLA:**

- 👨‍💻 **أنت:** المخترق (تلاعبت بالـ ID).
    
- 🌐 **المطعم (الموظف):** هو الـ API (نفذ الطلب بناءً على الرقم فقط).
    
- ⚠️ **المشكلة:** الموظف وثق في "الرقم" الذي أعطيته إياه ولم يتأكد هل أنت فعلاً صاحب الطلب رقم 8 أم لا.

## 🧩 3. أشهر الأنواع (BOLA Variations)

### 🔢 1. التلاعب بالمعرفات الرقمية (Simple Numeric IDs)

هذا هو النوع الأسهل والأكثر شيوعاً. المعرفات تكون عبارة عن أرقام متسلسلة.

- **المثال:** `GET /api/orders/1001` ← يغيرها الهكر إلى `1002`.
    
- 🎯 **الهدف:** سحب بيانات الطلبات لجميع العملاء عبر سكربت بسيط.

---

### 🧬 2. التلاعب بالـ UUIDs/GUIDs (The "Hidden" ID)

يعتقد المبرمج أن استخدام `UUID` يحمي من BOLA لأن تخمينه مستحيل.

- ⚠️ **الثغرة:** الهكر لا يخمن الـ UUID، بل يبحث عنه في أماكن أخرى مثل:
    
    - روابط البروفايل العامة
        
    - استجابات الـ API
        
    - ملفات JavaScript
        
    - سجلات أو تسريبات قديمة

- بمجرد حصوله على UUID صالح، يبدأ باستخدامه على Endpoints حساسة.

---

### 🔗 3. التلاعب بمصادر البيانات المتعددة (Multi-Object BOLA)

أحياناً الـ API يتأكد من صاحب الحساب، لكنه ينسى التأكد من "الأشياء" التابعة للحساب.

- **المثال:** حسابك (User 5) يطلب عرض فاتورة (Invoice 99).
    
- ⚠️ **المشكلة:** الـ API يتأكد أنك User 5، لكنه لا يتأكد أن Invoice 99 تتبع لك فعلاً.

---
